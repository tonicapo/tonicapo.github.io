Ecran :

Résolution physique de l'écran.

Commande Javascript : screen.width et screen.height

Ces commandes doivent normalement retourner la résolution d'affichage de l'écran en pixels physiques (et non en dips).

Cependant, cela varie en fonction des navigateurs et des périphériques. Avec Chrome 25 sur un galaxy Note, les valeurs retournées sont exprimées en dips, mais pas avec Firefox.

Pour information : px = dips x (dpi / 160)

Navigateur :

Dimensions totales du navigateur, incluant les barres d'outils, ascenceurs, etc.

Commande Javascript : window.outerWidth et window.outerHeight

Visual viewport :

Dimensions de l'écran en pixels css.

Commande Javascript : window.innerWidth et window.innerHeight

Il s'agit des dimensions, en pixels CSS, du contenu affiché sur la page. L'utilisateur peut scroller pour modifier la partie de la page affichée, ou zommer pour changer la taille du visual viewport.

Les valeurs initiales renvoyées par les commandes javascript peuvent dépendre du périphérique utilisé, du navigateur, etc.

Layout viewport :

Dimensions du gabarit de page de nase. Cette valeur est constante, quel que soit le niveau de zoom.

Commande Javascript : document.documentElement.clientWidth et document.documentElement.clientWidth

Les valeurs CSS, notamment les dimensions en pourcentage, sont calculées par rapport aux dimensions du Layout viewport. Ce comportement à été défini pour faire en sorte que la mise en page du site, par défaut, corresponde à la mise en page définie pour l'affichage sur pc.

Ses dimensions initiales diffèrent en fonction des navigateurs. Safari iPhone utilise une valeur de 980px, Opera 850px, Android Webkit 980px et IE 974px.

On peut le forcer Layout Viewport à avoir une largeur égale à celle du Visual viewport en utilisant la balise meta :

<meta name="viewport" content="width=device-width">

Densité de pixels :

Commande Javascript : window.devicePixelRatio

Densité de pixels du périphérique. Sur un écran rétina, et de plus en plus de périphériques, la valeur renvoyée est supérieure à 1 (plus de 1 pixel physique par pixel d'affichage).
